Shadow Maze Game Document

HongZhong Hu, Li Xing Liu, Yoyo He

Introduction

This document describes a game called “Shadow Maze”, which is designed to be a simple, 2D top-down game. The game will employ sprite-based animation, tiled background, collision detection, physics, AI, efficient memory management, render threading, and more basic 2D game techniques.

Technology

Shadow Maze will be developed for the Windows/macOS Platform using the Wolfie 2D game engine, which is a bare-bones engine developed by Joe Weaver and Professor Richard McKenna for rapid prototyping 2D games. The artwork will be simple, and so we’ll simply use Piskel.

Backstory

It all began with a deadly virus that ravaged the world, causing chaos and devastation. Not only did it kill all humans infected, it also caused mutations in animals, making them extremely aggressive and dangerous. The survivors of a kingdom moved underground, and built a series of mazes to protect themselves from the virus and the mutated creatures that roamed above. Years passed by, and people longed to know if it was safe to return to the surface, but they had forgotten the path through the mazes by then. The prince, a brave and adventurous young man, decided to venture through the mazes to reach the surface. However, they were no longer the peaceful places they had once been. Could the prince make it out alive?

Objective

The prince’s goal is to find his way through the dangerous underground labyrinths, and return to the surface in order to save humanity. The mazes are dark and full of dead ends, so the prince must be able to find the exits with limited light sources. Not only that, there are many dangerous beasts and creatures lurking in the shadows, so the prince must defeat all that comes his way, including a scary boss monster at the exit of each maze!

GamePlay

The game will work like a top down game, with the prince being able to run up, down, left, right through various paths available to him. The prince will have to be cautious because there are many traps throughout his journey to the outside. The prince will be mentally exhausted as not all the paths will be friendly to him. Some paths may provide hope and some may provide suffering. The prince will only have a limited view of his paths, but with a torch, the prince can increase his radius of light to allow him to see more. If the prince is lucky enough, he might find a treasure chest containing items that will benefit his journey through the Shadow Mazes. Some of the items can be a lantern or a light stone. The prince has to use his items carefully, because they will be essential for his survival and journey. The prince’s basic attack will be with a sword, dealing damage to nearby enemies. He has an ultimate ability in the form of a flying sword slash, which requires energy. The prince has an energy bar that can be charged with each basic attack on an enemy, and using his ultimate requires the energy bar to be full. The prince will have to carefully decide when to use his ultimate, since it will be crucial to his survival in the Shadow Mazes. In order to protect himself from the various traps and monsters throughout the maze, the prince carries a shield. With the shield, the prince can block attacks, and if timed correctly, he has the ability to parry, reflecting ranged attacks and stunning enemies with melee attacks. With each parry, the prince is able to charge his energy bar by a great amount.

Controls

This game will be played using a keyboard. Once started, use the following:

  • W - Move Up
  • A - Move Left
  • S - Move Down
  • D - Move Right
  • J - Attack
  • K - Shield
  • U - Ultimate (Sword Slash)
  • ESC - This pauses the game and presents the in-game menu, allowing the user to exit to the splash screen, start over, or get help. If a game is not in progress, ESC does nothing.

Grpahical User Interface

  • Splash Screen - The splash screen GUI simply presents the game logo and a Start button for the user to press when they are ready to play. Upon pressing it, the user will be brought to the main menu.


  • Main Menu - The game logo is presented, along with the following options for the player to choose from:
    • Select Level - If pressed, a list of available levels will pop up. If the user clicks on one of them, the user enters the level and starts the game.
    • Display Controls - If pressed, the game will display an info screen that includes a description of all game controls.
    • Help - If pressed, the game will display an info screen explaining important mechanics of the game (e.g. items, vision).
    • Exit - If pressed, the game program will exit.


  • In-Game Menu - While the game is in progress and ESC or the pause button is pressed, a game menu will pop up, allowing the player to select from the following options:
    • Restart - If pressed, a confirmation message will pop up, asking if the user really wishes to restart the level and lose all progress. If yes, the level is reset.
    • Select Level - If pressed, a list of available levels will pop up. If the user clicks on one of them, a confirmation message will pop up, asking if the user really wishes to restart the level and lose all progress. If yes, the level is reset.
    • Display Controls - Same as in Main Menu.
    • Help - Same as in Main Menu.
    • Exit - If pressed, a confirmation message will pop up, asking the user if they really wish to exit and lose all progress. If yes, the game program will exit to the main menu.


  • In-Game GUI - Once a level starts, the player spawns into the maze as the prince, with his sword and shield at the ready. While playing the following, will be displayed at all times:
    • Time Spent
    • Health Bar
    • Energy Bar
    • Pause Button

Artwork

All artwork in the game will be original. The following needs to be created:

  • The Prince - a simple sprite with attack, defend, and walking animation
    • Idle
    • Walking Right
    • Walking Left
    • Walking Up
    • Walking Down
    • Taking Damage
    • Attacking
    • Sword Slash
    • Defending/Parry
    • Dying
  • Monsters - simple sprites, each with their own unique animation
    • Idle
    • Walking Right
    • Walking Left
    • Walking Up
    • Walking Down
    • Taking Damage
    • Attacking
    • Dying
  • Floor and Wall Tiles - Shadow Maze will have 6 Levels, each with their own unique style. The floor tiles will clearly define what areas are walkable and it will be clear which are wall tiles
  • Fake Wall Tile - A tile that looks like a Wall tile, but is able to walk through it
  • Traps - A simple sprite that will cause damage when it is stepped over by character.
  • Treasure Chest - a simple sprite that can be opened when walked over, with different possible items inside for the character

Sound Effects

All sound effects will be original. Sounds must be made to coincide with each of the following events:

  • Walking
  • Attacking
  • Shielding
  • Parrying
  • Healing
  • Taking Damage
  • Dying
  • Charging Sword Slash
  • Opening Treasure Chest
  • Monster Dying
  • Clearing Level

Music

Music will be created using Audacity, and every level will feature background music to liven the atmosphere of the game.
